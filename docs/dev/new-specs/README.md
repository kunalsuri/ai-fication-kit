# New Specs

This directory holds **specification documents that are under development** and
have not yet been implemented.

Once a spec has been fully implemented and verified, move it to
[`../specs-implemented/`](../specs-implemented/).

## Guidelines

- One Markdown file per feature/change.
- Follow the license header convention (see the HTML comment at the top of this
  file).
- Keep specs self-contained: include motivation, design, API surface, and a
  verification plan so reviewers and AI agents can audit them independently.
