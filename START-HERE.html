Start here — run ai-fication-kit

Give your AI coding agent a trusted map of any existing or legacy repo. Copy a command below into your terminal. Replace /path/to/your/repo with the folder you want to make AI-native.

Apache 2.0 Node 18+ Python 3.8+ Zero dependencies
🛡️ Safe: the installer only copies plain-text files into the folder you choose — it runs no commands and makes no network calls. Add --dry-run to any install command to preview without writing.

1 Scaffold the kit into your repo

Pick one. The npx way needs nothing cloned; the clone way runs the pure Node or Python installer.

# A · Quickest — straight from GitHub, nothing to clone (needs Node 18+)
# Preview first (writes nothing):
npx github:kunalsuri/ai-fication-kit shazam /path/to/your/repo --dry-run
# Then install for real:
npx github:kunalsuri/ai-fication-kit shazam /path/to/your/repo
# B · From a local clone — pure Node or Python stdlib (no npm/pip install)
git clone https://github.com/kunalsuri/ai-fication-kit.git
cd ai-fication-kit
node install.mjs shazam /path/to/your/repo      # Node 18+
python install.py shazam /path/to/your/repo     # …or Python 3.8+

2 Draft the maps

Open your repo in Claude Code (or your agent of choice) and run the cold-start command. It scans the code and drafts the map (~5 min), all tagged [inferred].

/cold-start

Using Cursor / Copilot / Codex? They read AGENTS.md — paste .claude/commands/cold-start.md as a prompt to run the same pass.

3 Audit, then keep it honest

Open ai/guide/MODULE_MAP.md, set each module's Stability, and flip confirmed rows from [inferred] to [verified]. Then cross-check every file-path claim against the real tree (deterministic, no LLM):

node install.mjs verify /path/to/your/repo       # or: python install.py verify ...
# add --strict to fail (exit 1) on stale claims, e.g. in CI

Build features safely with /add-feature. Full walkthrough: the repo's README.md and docs/GETTING-STARTED.md.
