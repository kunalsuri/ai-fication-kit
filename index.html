Give your AI coding agent a trusted map of any existing or legacy repo. One command scaffolds a compact, provenance-tracked knowledge layer — drafted by the agent, verified by you, and kept mechanically honest. This page explains the kit and hands you the exact command; the real work happens in a small, readable script on your own machine — not in this browser tab.

Apache 2.0 Node 18+ Python 3.8+ Zero dependencies Claude Code · Cursor · Copilot · Codex · Windsurf

Is this safe? Yes — here's why

The most important thing to understand before you run anything.

  • The installer only copies plain-text files (documentation & agent config) into a folder you choose. Text can't execute.
  • It runs no commands, makes no network connections, and has zero dependencies — Node stdlib or Python stdlib only.
  • It never writes outside the folder you point at, and never overwrites a file unless you pass --force.
  • Run it with --dry-run first to see the exact plan before a single byte is written.
  • It records every file it writes in ai/install-manifest.json, so uninstall can remove exactly what it added — no trace left.
  • The installer is split into small, single-purpose modules (lib/orient, lib/installer, lib/verify, lib/drift, lib/intake) — each auditable in one sitting. Open install.mjs or install.py and read it first. We want you to.
  • This web page cannot run the installer or open a terminal — browsers are sandboxed to prevent exactly that. That's a safety feature, not a limitation to work around.

What the kit does, in six steps

The script does the deterministic parts (orient, install, verify). Your agent drafts. You do the ~30-minute audit that makes the map trustworthy.

1 · orient
Reads marker files (package.json, pom.xml, pyproject.toml…) → ai/repo-profile.json. Deterministic, no LLM.
2 · install
Stamps the kit's templates into your repo. Steps 1+2 are the shazam one-shot.
3 · /cold-start
Your agent reads the code and drafts MODULE_MAP + diagrams — everything tagged [inferred].
4 · Your audit ⚠
Set each module's Stability (frozen / stable / ours / ?) and flip [inferred]→[verified]. ~30 min.
5 · verify
Mechanically checks every file-path claim in the docs against the real tree. Deterministic, no LLM.
6 · /add-feature
The agent builds against the trusted map, never touching frozen code, keeping token use low.
Script (deterministic) AI agent (inferred) You (the human gate)

After installing

    
In your repo, open Claude Code (or your agent of choice) and run:

    
cd /path/to/your/repo && claude

    
/cold-startdraft the maps (~5 min)

    
Then do the ~30-minute audit in ai/guide/MODULE_MAP.md, optionally run verify to keep the docs honest, and build features with /add-feature. Using Cursor / Copilot / Codex? They read AGENTS.md; paste .claude/commands/cold-start.md as a prompt to drive the same pass. Full walkthrough lives in the repo's README.md and docs/.

  


  
  

    
What lands in your repo

    
A minimal, structured knowledge layer the agent reads instead of crawling your whole tree.

    
your-repo/
├── CLAUDE.md                ← auto-loaded by Claude Code (thin; points everywhere else)
├── AGENTS.md                ← same rules for Cursor, Copilot, Codex, Windsurf
├── ai/
│   ├── INDEX.md             role → path manifest
│   ├── repo-profile.json    machine-readable facts from orient
│   ├── install-manifest.jsonwhat the installer wrote (for clean uninstall)
│   ├── guide/               navigation, loaded every session
│   │   ├── MODULE_MAP.md    directory → responsibility → Stability  ← START HERE
│   │   └── PROJECT_OVERVIEW · ARCHITECTURE · FEATURE_MAP · CONVENTIONS
│   ├── analysis/            FEATURE_CATALOG, diagrams, audit-reports, problems
│   └── lab/                 specs, decisions (ADRs), evaluations, experiments
└── .claude/                 commands, subagents, the add-feature skill
